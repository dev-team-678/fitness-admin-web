## 一、项目概述

### 1.1 产品定位

运动健身助手小程序的配套管理后台，面向运营人员和管理员，提供用户管理、内容管理、数据监控、**AI 知识库运营、对话质量监控、智能计划管理**等功能，支撑小程序的日常运营和业务决策。

### 1.2 目标用户

| 角色 | 职责 | 核心需求 |
|------|------|---------|
| 超级管理员 | 系统配置、权限管理 | 账号管理、角色权限、系统参数 |
| 内容运营 | 训练计划/动作库维护 | 计划编辑、动作上传、上下架管理 |
| 数据运营 | 数据分析与监控 | 用户增长、活跃度、训练数据看板 |
| 社区管理 | 社区内容审核 | 动态审核、评论管理、举报处理 |
| **AI 运营** 🆕 | **AI 知识库维护、对话质量监控** | **知识库 CRUD、对话审核、安全规则配置** |

### 1.3 核心价值

- **内容管理**：可视化管理训练计划、动作库等核心内容
- **用户洞察**：全面的用户数据统计与行为分析
- **运营支撑**：成就系统配置、推送管理、活动运营
- **风险管控**：社区内容审核、异常行为监控
- **AI 智能运营** 🆕：知识库管理、对话质量保障、AI 计划审核、安全规则配置

### 1.4 技术栈说明

| 层级 | 选型 | 说明 |
|------|------|------|
| 前端框架 | **Vue 3.4+ (Composition API)** | `<script setup>` + TypeScript |
| UI 组件库 | **Element Plus 2.x** | Vue 3 原生支持，表格/表单/树/Upload 组件完善 |
| 构建工具 | **Vite 5** | 开发热更新秒级，构建产物小 |
| 状态管理 | **Pinia** | Vue 3 官方推荐，TypeScript 友好 |
| 路由 | **Vue Router 4** | 路由守卫 + 动态路由（RBAC 权限注入） |
| HTTP 客户端 | **Axios** | 拦截器统一处理认证/错误 |
| 图表库 | **ECharts 5** | 趋势图、热力图、饼图、漏斗图 |
| Markdown 编辑器 | **md-editor-v3** | Vue 3 原生 Markdown 编辑器 |
| 后端 | **Java 17 + Spring Boot 3.2** | 详见后端 PRD |
| 数据库 | **MySQL 8.0+** | JSON 索引、窗口函数、FULLTEXT 支持 |
| 向量数据库 | **Milvus 2.x** | RAG 语义检索 |

| 新用户次日留存 | 注册次日有行为的用户/当日注册用户 | user.created_at + 活跃表 |
| **AI 对话满意度** 🆕 | **有用反馈数 / 总反馈数** | **ai_chat_message.feedback** |
| **AI 计划确认率** 🆕 | **confirmed 数 / draft 数** | **ai_plan.status** |

#### 3.10.2 SSE 实时对话流（P2）

**技术选型**：SSE（Server-Sent Events）替代 WebSocket，降低实现和运维成本。

| 维度 | WebSocket | SSE (✅ 选定) |
|------|-----------|--------------|
| 实现复杂度 | 高（双向协议、心跳、断线重连） | **低（HTTP 单向推送）** |
| 运维成本 | 高（需独立端口/负载均衡） | **低（复用 HTTP 连接）** |
| 自动重连 | 需手动实现 | **浏览器原生支持** |
| 数据方向 | 双向 | **服务端→客户端（足够）** |

**前端实现**：

```typescript
// composables/useChatStream.ts
export function useChatStream(sessionId: number) {
  const messages = ref<ChatMessage[]>([])
  const status = ref<'connecting' | 'connected' | 'disconnected'>('disconnected')
  let eventSource: EventSource | null = null

  function connect() {
    status.value = 'connecting'
    eventSource = new EventSource(`/api/admin/v1/ai/chat-sessions/${sessionId}/stream`)
    eventSource.onmessage = (event) => {
      messages.value.push(JSON.parse(event.data))
    }
    eventSource.onopen = () => { status.value = 'connected' }
    eventSource.onerror = () => {
      status.value = 'disconnected'
      // EventSource 自动重连，3s 后重试
    }
  }

  onUnmounted(() => { eventSource?.close() })
  return { messages, status, connect }
}
```

**降级方案**：SSE 连接失败时，自动降级为 10 秒间隔的 HTTP 轮询。

#### 3.15.2 管理员独立表 🆕

**设计原则**：独立 `admin_user` 表，不侵入小程序 `user` 表，通过 `user_id` 关联。

```sql
CREATE TABLE `admin_user` (
  `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
  `user_id` BIGINT UNSIGNED NOT NULL COMMENT '关联小程序 user.id',
  `username` VARCHAR(64) NOT NULL COMMENT '管理员登录用户名',
  `password_hash` VARCHAR(256) NOT NULL COMMENT '密码哈希(bcrypt)',
  `role_id` INT UNSIGNED DEFAULT NULL COMMENT '角色ID',
  `status` TINYINT(1) NOT NULL DEFAULT 1 COMMENT '1-正常 2-禁用',
  `last_login_at` DATETIME DEFAULT NULL,
  `last_login_ip` VARCHAR(45) DEFAULT NULL,
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_username` (`username`),
  UNIQUE KEY `uk_user_id` (`user_id`),
  CONSTRAINT `fk_admin_user` FOREIGN KEY (`user_id`) REFERENCES `user`(`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='管理员账号';
```

- **Base URL**: `/api/admin/v1`

```json
{
  "code": 0,

}
```

### 6.2 Axios 封装

```typescript
// api/request.ts
const service = axios.create({
  baseURL: '/api/admin/v1',
  timeout: 30000,
})

// 请求拦截：自动注入 JWT
service.interceptors.request.use((config) => {
  const token = useUserStore().token
  if (token) config.headers.Authorization = `Bearer ${token}`
  return config
})

// 响应拦截：统一错误处理
service.interceptors.response.use(
  (response) => {
    const { code, message, data } = response.data
    if (code !== 0) {
      ElMessage.error(message)
      return Promise.reject(new Error(message))
    }
    return response.data
  },
  (error) => {
    if (error.response?.status === 401) {
      useUserStore().logout()
      router.push('/login')
    }
    return Promise.reject(error)
  }
)
```

### 6.3 路由权限控制

```typescript
// router/guards.ts
router.beforeEach(async (to, from, next) => {
  const userStore = useUserStore()
  const permStore = usePermissionStore()

  if (!userStore.token) {
    if (to.path === '/login') return next()
    return next('/login')
  }

  // 首次加载：获取用户信息 + 生成动态路由
  if (!permStore.routesGenerated) {
    await userStore.getUserInfo()
    const routes = await permStore.generateRoutes(userStore.permissions)
    routes.forEach(r => router.addRoute(r))
    return next({ ...to, replace: true })
  }

  // 权限校验
  if (to.meta.permission && !permStore.hasPermission(to.meta.permission)) {
    return next('/403')
  }
  next()
})
```

### 6.4 状态管理 (Pinia)

```typescript
// stores/user.ts
export const useUserStore = defineStore('user', () => {
  const token = ref<string>('')
  const userInfo = ref<AdminUser | null>(null)
  const permissions = ref<string[]>([])

  async function login(username: string, password: string) {
    const res = await authApi.login({ username, password })
    token.value = res.data.token
    // Token 持久化到 localStorage
  }

  async function getUserInfo() {
    const res = await authApi.profile()
    userInfo.value = res.data
    permissions.value = res.data.permissions
  }

  function logout() {
    token.value = ''
    userInfo.value = null
    permissions.value = []
  }

  return { token, userInfo, permissions, login, getUserInfo, logout }
})
```

### 6.5 COS 直传上传

```typescript
// composables/useCosUpload.ts
export function useCosUpload() {
  const uploading = ref(false)
  const progress = ref(0)

  async function upload(file: File, dir: string): Promise<string> {
    uploading.value = true
    // 1. 后端获取临时签名 URL
    const { data } = await api.getUploadUrl({ filename: file.name, dir })
    // 2. 前端直传 COS
    await axios.put(data.uploadUrl, file, {
      headers: { 'Content-Type': file.type },
      onUploadProgress: (e) => { progress.value = Math.round(e.loaded / e.total * 100) }
    })
    uploading.value = false
    return data.fileUrl
  }

  return { upload, uploading, progress }
}
```

```

│  ┌──────────┐              ┌──────────┐                      │
│  │  plan    │              │admin_user│ ← 管理员独立表        │
│  │ exercise │  ← 共享表 →  │admin_role│   通过 user_id 关联   │

│                            │ai_prompt*│                      │
│                            └──────────┘                      │
└─────────────────────────────────────────────────────────────┘
```

**关键约束**：
- **admin_user 独立**：管理员账号、密码、角色完全独立于小程序 user 表
- **user_id 关联**：admin_user.user_id → user.id，管理员同时也是小程序用户
- **AI 表共享**：ai_chat_session、ai_chat_message、ai_plan、knowledge_base 等由小程序写入、管理后台读写
- **管理后台独有表**：admin_user、admin_role、admin_operation_log、ai_safety_rule、ai_prompt_template 等在同一个数据库中
- 建议使用只读副本 + 主库写入架构，避免影响小程序性能
- 所有管理后台写操作记录操作日志
